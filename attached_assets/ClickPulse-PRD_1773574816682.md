# ClickPulse — Mouse Activity Tracker
## PRD Completo para Build

---

## 1. Visão Geral

**ClickPulse** é um aplicativo desktop Windows que monitora atividade do mouse em tempo real. Rastreia cliques por hora, tempo ativo vs pausas, tipos de clique (esquerdo/direito/meio) e mantém histórico diário/semanal com dashboard visual e exportação CSV.

**Plataforma:** Windows Desktop (app local)
**Usuário:** Uso pessoal, monitoramento de produtividade

## 4. Schema do Banco de Dados (SQLite)

### Tabela: clicks
Cada clique individual registrado.

```sql
CREATE TABLE clicks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME NOT NULL DEFAULT (datetime('now', 'localtime')),
    button TEXT NOT NULL CHECK(button IN ('left', 'right', 'middle')),
    x INTEGER NOT NULL,
    y INTEGER NOT NULL
);
CREATE INDEX idx_clicks_timestamp ON clicks(timestamp);
```

### Tabela: activity_periods
Períodos contínuos de atividade ou pausa.

```sql
CREATE TABLE activity_periods (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    start_time DATETIME NOT NULL,
    end_time DATETIME,
    type TEXT NOT NULL CHECK(type IN ('active', 'pause'))
);
CREATE INDEX idx_activity_start ON activity_periods(start_time);
```

### Tabela: hourly_stats
Agregação por hora — gerada automaticamente.

```sql
CREATE TABLE hourly_stats (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    hour_start DATETIME NOT NULL UNIQUE,
    total_clicks INTEGER NOT NULL DEFAULT 0,
    left_clicks INTEGER NOT NULL DEFAULT 0,
    right_clicks INTEGER NOT NULL DEFAULT 0,
    middle_clicks INTEGER NOT NULL DEFAULT 0,
    active_seconds INTEGER NOT NULL DEFAULT 0,
    pause_seconds INTEGER NOT NULL DEFAULT 0
);
CREATE INDEX idx_hourly_start ON hourly_stats(hour_start);
```

### Tabela: settings
Configurações do usuário (chave-valor, JSON-encoded).

```sql
CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
```

### Settings padrão:
```json
{
    "pause_threshold_minutes": 3,
    "console_refresh_seconds": 30,
    "activity_check_seconds": 10,
    "notification_click_milestone": 100,
    "long_pause_alert_minutes": 30
}
```

## 5. Módulos — Especificação Detalhada

### 5.1 database.py

Classe `Database`:
- `__init__(db_path="clickpulse.db")` → abre/cria banco, PRAGMA WAL, row_factory=Row
- Usa `threading.Lock` em TODOS os métodos (pynput roda em thread separada)
- `check_same_thread=False` na conexão
- `_init_tables()` → CREATE TABLE IF NOT EXISTS para todas as 4 tabelas + índices
- `insert_click(button, x, y) → int` — insere clique com timestamp=now
- `get_clicks_since(since: str) → list[dict]`
- `get_clicks_in_range(start, end) → list[dict]`
- `count_clicks_in_range(start, end) → dict` — retorna {total, left_clicks, right_clicks, middle_clicks}
- `insert_activity_period(start_time, type_, end_time=None) → int`
- `close_activity_period(period_id, end_time)`
- `get_open_period() → dict|None` — período sem end_time
- `get_activity_periods_in_range(start, end) → list[dict]`
- `upsert_hourly_stats(hour_start, total, left, right, middle, active_sec, pause_sec)`
- `get_hourly_stats(date_str) → list[dict]` — stats de todas as horas de um dia
- `get_setting(key, default=None) → str|None`
- `set_setting(key, value)`
- `close()`

### 5.2 config.py

Classe `Config`:
- `__init__(database)` → garante que defaults existem no banco
- `get(key, default) → Any` — lê do banco, JSON decode
- `set(key, value)` — JSON encode, salva no banco
- Properties: `pause_threshold_minutes`, `activity_check_seconds`
- DEFAULTS dict com todos os valores padrão

### 5.3 tracker.py

Classe `MouseTracker`:
- `__init__(database, on_click_callback=None)`
- `start()` → inicia `pynput.mouse.Listener` como thread daemon
- `stop()` → para o listener
- `_on_click(x, y, button, pressed)`:
  - Se pressed=True: mapear Button.left→'left', Button.right→'right', Button.middle→'middle'
  - `database.insert_click(button_str, x, y)`
  - Atualizar `self._last_activity = datetime.now()`
  - Chamar `on_click_callback(button_str, x, y)` se existir
- `_on_move(x, y)` → atualizar `self._last_activity` (NÃO salva no banco, só marca atividade)
- `_on_scroll(x, y, dx, dy)` → atualizar `self._last_activity`
- Property `last_activity_time → datetime`
- Property `is_running → bool`

**IMPORTANTE:** Qualquer movimento do mouse, clique ou scroll conta como atividade.

### 5.4 activity.py

Classe `ActivityDetector`:
- `__init__(database, config, tracker)`
- `start()` → inicia thread daemon com `_check_loop()`
- `stop()` → seta flag `_running = False`
- `_check_loop()`:
  - A cada `config.activity_check_seconds` (padrão 10s):
  - Calcular `idle_time = now - tracker.last_activity_time`
  - Se `idle_time >= pause_threshold` E estado='active':
    - Fechar período ativo (end_time = tracker.last_activity_time)
    - Abrir período de pausa (start_time = tracker.last_activity_time)
    - Estado → 'pause'
  - Se `idle_time < pause_threshold` E estado='pause':
    - Fechar período de pausa (end_time = now)
    - Abrir período ativo (start_time = now)
    - Estado → 'active'
  - Recarregar `pause_threshold` do config a cada check (permite mudança em runtime)
- Property `current_state → 'active'|'pause'`
- Property `active_time_today → timedelta` (soma dos períodos 'active' de hoje)
- Property `pause_time_today → timedelta` (soma dos períodos 'pause' de hoje)
- Na inicialização: abrir período 'active' automaticamente

### 5.5 aggregator.py

Classe `Aggregator`:
- `__init__(database)`
- `aggregate_hour(hour_start: datetime)`:
  - Calcular intervalo [hour_start, hour_start + 1h)
  - Contar cliques por tipo com `count_clicks_in_range()`
  - Calcular active_seconds e pause_seconds dos activity_periods que intersectam a hora
  - `upsert_hourly_stats()` no banco
- `get_hourly_stats(date: str) → list[dict]`
- `get_current_hour_clicks() → int` — contagem em tempo real da hora atual
- `aggregate_previous_hour()` — helper que calcula hour_start da hora anterior e agrega

### 5.6 notifier.py

Classe `Notifier`:
- Usa notificações nativas do Windows (win10toast ou plyer)
- `notify(title, message, duration=5)`
- `check_milestones(total_clicks_today, config)`:
  - Se total_clicks % milestone == 0 → notificar "🎯 X cliques hoje!"
- `check_long_pause(pause_duration, config)`:
  - Se pausa > long_pause_alert_minutes → notificar "⏸ Pausa longa detectada"
- `hourly_summary(hour_stats)`:
  - Notificar resumo: "⏰ Última hora: X cliques, Y min ativo"

### 5.7 exporter.py

Classe `Exporter`:
- `export_day_csv(database, date_str, output_path)`:
  - Colunas: timestamp, button, x, y, clicks_per_hour, active_minutes, pause_minutes
  - Inclui cliques individuais + resumo horário ao final
- `export_range_csv(database, start_date, end_date, output_path)`:
  - Mesmo formato, múltiplos dias
- Usa módulo `csv` padrão do Python

### 5.8 ui/main_window.py

Classe `MainWindow(QMainWindow)`:
- Título: "ClickPulse — Mouse Activity Tracker"
- Tamanho: 900x600, mínimo 700x500
- **3 tabs (QTabWidget):**
  1. Dashboard (padrão, abre nesta)
  2. Histórico
  3. Configurações
- Toolbar com botões: Pausar/Retomar rastreamento, Exportar CSV
- Status bar mostrando: estado atual (●Ativo / ●Pausa) + cliques/hora
- Minimizar para tray ao fechar (não encerrar app)
- Ícone do app na title bar e taskbar

### 5.9 ui/dashboard.py

Classe `DashboardWidget(QWidget)`:
- **Painel de resumo** (topo, cards lado a lado):
  - 🖱 Cliques hoje: XXXX
  - ⏱ Tempo ativo: Xh Xm
  - ⏸ Pausas: Xh Xm
  - 📊 Cliques/hora atual: XX
- **Gráfico de barras** (pyqtgraph BarGraphItem):

  - Eixo Y: total de cliques
  - Barras coloridas, hora atual destacada
- **Gráfico de pizza** (QPainter custom ou pyqtgraph):
  - Distribuição: esquerdo (azul), direito (verde), meio (laranja)
  - Legenda com porcentagens
- **Timeline de atividade** (pyqtgraph horizontal):
  - Barra horizontal representando o dia

### 5.10 ui/history.py

Classe `HistoryWidget(QWidget)`:

## 6. main.py — Integração

```python
"""ClickPulse — Entry point."""

import sys
import signal
from datetime import datetime, timedelta
from PyQt6.QtWidgets import QApplication
from PyQt6.QtCore import QTimer

from clickpulse.database import Database
from clickpulse.config import Config
from clickpulse.tracker import MouseTracker
from clickpulse.activity import ActivityDetector
from clickpulse.aggregator import Aggregator
from clickpulse.notifier import Notifier
from clickpulse.ui.main_window import MainWindow
from clickpulse.ui.tray import SystemTray


def main():
    app = QApplication(sys.argv)
    app.setQuitOnLastWindowClosed(False)  # Continua no tray

    # Core
    db = Database("clickpulse.db")
    config = Config(db)
    notifier = Notifier()
    aggregator = Aggregator(db)

    # Tracker com callback para UI
    tracker = MouseTracker(db)
    activity = ActivityDetector(db, config, tracker)

    # UI
    window = MainWindow(db, config, tracker, activity, aggregator)
    tray = SystemTray(window, tracker, activity)

    # Start tracking
    tracker.start()
    activity.start()

    # Timer: agregar hora anterior a cada hora cheia
    hourly_timer = QTimer()
    hourly_timer.timeout.connect(aggregator.aggregate_previous_hour)
    hourly_timer.start(60_000)  # Check a cada minuto

    # Timer: refresh dashboard
    refresh_timer = QTimer()
    refresh_timer.timeout.connect(window.refresh_dashboard)
    refresh_timer.start(5_000)  # 5 segundos

    # Timer: notificações
    notify_timer = QTimer()
    def check_notifications():
        today = datetime.now().strftime("%Y-%m-%d")
        clicks = db.get_clicks_since(today)
        notifier.check_milestones(len(clicks), config)
        if activity.current_state == 'pause':
            notifier.check_long_pause(activity.current_pause_duration, config)
    notify_timer.timeout.connect(check_notifications)
    notify_timer.start(30_000)

    window.show()
    tray.show()

    # Graceful shutdown
    def shutdown():
        tracker.stop()
        activity.stop()
        aggregator.aggregate_previous_hour()
        db.close()

    app.aboutToQuit.connect(shutdown)
    signal.signal(signal.SIGINT, lambda *_: app.quit())

    sys.exit(app.exec())


if __name__ == "__main__":
    main()
```

## 8. Regras de Negócio

1. **Atividade**: qualquer evento de mouse (move, click, scroll) reseta o timer de inatividade
2. **Pausa**: se nenhum evento por >= `pause_threshold_minutes` → período muda para pausa
3. **Retorno**: qualquer evento após pausa → fecha período de pausa, abre novo período ativo
4. **Agregação**: a cada hora cheia, calcular e salvar stats no hourly_stats
5. **Thread safety**: pynput roda em thread separada — todo acesso ao banco usa Lock
6. **Fechar janela**: minimiza para tray (não encerra). Sair só pelo menu do tray
7. **Startup**: ao iniciar, abrir período 'active' automaticamente
8. **Shutdown**: ao encerrar, fechar período aberto, rodar agregação final, fechar banco
